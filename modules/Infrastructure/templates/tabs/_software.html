Software Info

Usage guide, fields, recipes and install modes.

Workflow
  1. Add a package in the Add Package tab.
  2. Define the all-users install recipe and optional user-scoped recipe.
  3. Select a package in Package Library.
  4. Choose targets: checked nodes or a group.
  5. Choose Install or Uninstall, select scope, and dispatch the task.
  6. Review execution logs in Queue.
Package Fields

Name / Version / Vendor identify the software in the library and task logs.

Package type controls ZIP extraction behavior and documents installer type.

Architecture marks x64, x86, ARM64, or any.

File uploads the installer to WinHUB. SHA256 is calculated automatically.

External URL points agents to another repository. Provide SHA256 for safety.

Install Recipes

All users install command is used for machine-wide installs.

Specific users install command is used only when you select the specific-users mode.

Uninstall command is used by the Uninstall action from the Software panel.

Available variables: {file}, {extract_dir}, {package_dir}, {users}, {name}, {version}.

Detection And Exit Codes

Detection type tells WinHUB how to confirm the software is installed.

Detection value is a path, registry key, or command depending on detection type.

Expected exit codes are successful installer codes, usually 0,3010.

If detection already matches, the generated task skips installation.

Edit And Delete

Hover a package card and click Edit to update metadata, recipes, detection rules, or replace the uploaded file.

Use Remove Current File only when you also provide a replacement file or an external URL.

Uninstall runs the saved uninstall command on selected hosts. Delete removes the package record and its uploaded file from the server.

Процес роботи
  1. Додай пакет у вкладці Add Package.
  2. Задай рецепт встановлення для всіх користувачів і, за потреби, рецепт для конкретних користувачів.
  3. Обери пакет у Package Library.
  4. Обери цілі: відмічені хости або групу.
  5. Обери Install або Uninstall, режим виконання і запусти задачу.
  6. Перевір лог виконання у Queue.
Поля пакета

Name / Version / Vendor показуються у бібліотеці та логах задач.

Package type описує тип інсталятора і керує розпаковкою ZIP.

Architecture позначає x64, x86, ARM64 або any.

File завантажує інсталятор на WinHUB. SHA256 рахується автоматично.

External URL дає агентам посилання на інше сховище. Для безпеки вкажи SHA256.

Рецепти встановлення

All users install command використовується для встановлення на весь сервер.

Specific users install command використовується лише в режимі конкретних користувачів.

Uninstall command використовується дією Uninstall у панелі Software.

Доступні змінні: {file}, {extract_dir}, {package_dir}, {users}, {name}, {version}.

Detection і коди виходу

Detection type визначає, як WinHUB перевіряє, що софт встановлено.

Detection value це шлях, registry key або команда, залежно від типу detection.

Expected exit codes це успішні коди інсталятора, зазвичай 0,3010.

Якщо detection вже спрацював, задача пропустить встановлення.

Редагування і видалення

Наведи на картку пакета і натисни Edit, щоб змінити параметри, рецепти, detection або замінити файл.

Remove Current File використовуй тільки якщо одразу додаєш новий файл або вказуєш external URL.

Uninstall запускає збережену команду видалення на вибраних хостах. Delete видаляє запис пакета і завантажений файл із сервера.
