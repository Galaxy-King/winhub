Action Script

Runs PowerShell or shell code on selected endpoints. Add variables as placeholders like {{ "{{admin_username}}" }}; WinHUB shows input fields before execution and API clients pass them in variables.

Metric Item

Runs as a reusable telemetry check. Return compact JSON or a scalar value; WinHUB stores the result as a host metric and can use it in trigger rules.

Report Template

Formats aggregated task results after execution. Use report templates for email bodies or review reports; they are selected in Post-Execution, not executed directly on hosts.

Variable pattern
$TargetUser = "{{ "{{admin_username}}" }}"
$TicketId = "{{ "{{ticket_id}}" }}"
$StaticPassword = "{{ "{{secret:local_admin_password}}" }}"

# API payload:
# "variables": { "admin_username": "admin", "ticket_id": "INC-1001" }
# Secrets are created in Template Secrets and are not sent by API clients.

Workspace Guide

Template authoring, execution rules, and sharing

How Templates Work

Action Script runs code on selected endpoints. For Windows agents this is PowerShell. For Linux/macOS-style agents or shell targets, use Bash/SH where supported by the agent.

Metric Item runs like an action, but the result is stored as a host metric and can be used by dashboards and trigger rules.

Report Template does not run on endpoints. It runs on the WinHUB server after tasks finish and formats aggregated task results for review or email.

Variables And Secrets

Use {{ "{{variable_name}}" }} for operator-provided values. WinHUB detects placeholders and asks for values before execution. Use {{ "{{secret:name}}" }} for values stored in Template Secrets; secret values are encrypted at rest and are injected only when the task is dispatched.

{% raw %}$User = "{{user_login}}"
$Password = "{{secret:temporary_password}}"
$VpnName = "{{vpn_name}}"{% endraw %}

Sensitive Output Masking

Reports are stored with the original body so secure email can include real secrets. Users without view_sensitive_reports see masked values in the UI. Avoid saving masked report text over the original. Password-like fields are masked by names and patterns such as password, temporary_password, secret, token, Pass:, and PASSWORD -.

Action Script Output

Return clear text for simple tasks. Return compact JSON when a Report Template must group, filter, or build emails. The server tries to parse each endpoint result as JSON and exposes it as result.data; plain text is exposed as result.log.

{% raw %}$result = @{
    winhub_report_type = "terminal_access_letters"
    hostname = $env:COMPUTERNAME
    users = @(
        @{ username = "rhein"; status = "success"; password = "..." }
    )
}
Write-Output ($result | ConvertTo-Json -Depth 8 -Compress){% endraw %}

Report Templates

Report templates use Jinja syntax and receive results and job_title. Each result has host, status, data, and log. Select a Report Template in Post-Execution before running an action.

{% raw %}{% for result in results %}
{{ result.host }} - {{ result.status }}
{{ result.data.message or result.log }}
{% endfor %}{% endraw %}

Split Reports

A report template can create several Reports from one task run. Wrap each report body in [[WINHUB_REPORT title="..."]] and [[/WINHUB_REPORT]]. This is useful for one letter per user, one report per customer, or one report per application.

{% raw %}{% for username in usernames %}
[[WINHUB_REPORT title="Access letter - {{ username }}"]]
Hello, {{ username }}.
Your personal data goes here.
[[/WINHUB_REPORT]]
{% endfor %}{% endraw %}

Windows Agent Execution Rules

  • Use PowerShell 5.1-compatible syntax unless you know all targets have newer PowerShell.
  • Agents run unattended, usually as a service account/SYSTEM, so no UI prompts, confirmations, or interactive installers.
  • Set $ErrorActionPreference = "Stop" for predictable failures and wrap risky blocks in try/catch.
  • Use UTF-8 console output when returning Ukrainian/Russian/English text.
  • Prefer idempotent scripts: repeated execution should not break existing state.
  • Use localized-safe SIDs for Windows groups when possible, for example Administrators S-1-5-32-544 and Remote Desktop Users S-1-5-32-555.
  • Use full paths for files and installers. Clean temporary files after use.
  • Return JSON only as the final output when a report template must parse it.

Do Not Do This

  • Do not use Read-Host, GUI installers, or prompts waiting for user input.
  • Do not print permanent passwords or tokens unless the report/email workflow requires it.
  • Do not run destructive commands without explicit target checks.
  • Do not depend on current working directory; agents may run from service/system paths.

Sharing And Permissions

Use Share with Team only for templates that are safe for operators. Operators also need access to target endpoint groups and run_tasks. Reports use separate permissions: view_reports, send_reports, edit_reports, delete_reports, and view_sensitive_reports.

Як працюють шаблони

Action Script запускає код на вибраних endpoint-ах. Для Windows Agent це PowerShell. Для Linux/macOS-style агентів або shell-цілей використовується Bash/SH, якщо це підтримує агент.

Metric Item запускається як action, але результат зберігається як метрика хоста і може використовуватися в dashboard та trigger rules.

Report Template не виконується на endpoint-ах. Він виконується на сервері WinHUB після завершення задач і форматує результат для перегляду або email.

Змінні і Secrets

Використовуй {{ "{{variable_name}}" }} для значень, які вводить оператор перед запуском. Використовуй {{ "{{secret:name}}" }} для значень із Template Secrets; вони зашифровані в сховищі і підставляються тільки під час dispatch задачі.

{% raw %}$User = "{{user_login}}"
$Password = "{{secret:temporary_password}}"
$VpnName = "{{vpn_name}}"{% endraw %}

Приховування паролів

Reports зберігаються з оригінальним тілом, щоб secure email міг відправити реальні секрети. Користувачі без права view_sensitive_reports бачать замасковані значення в UI. Маскуються назви і патерни: password, temporary_password, secret, token, Pass:, PASSWORD -.

Що повертати зі скрипта

Для простих задач повертай зрозумілий текст. Якщо Report Template має групувати, фільтрувати або формувати листи, повертай компактний JSON. Сервер пробує розпарсити результат кожного endpoint-а як JSON і передає його як result.data; звичайний текст доступний як result.log.

{% raw %}$result = @{
    winhub_report_type = "terminal_access_letters"
    hostname = $env:COMPUTERNAME
    users = @(
        @{ username = "rhein"; status = "success"; password = "..." }
    )
}
Write-Output ($result | ConvertTo-Json -Depth 8 -Compress){% endraw %}

Report Templates

Report templates використовують Jinja-синтаксис і отримують results та job_title. Кожен result має host, status, data, log. Обирай Report Template у Post-Execution перед запуском action.

{% raw %}{% for result in results %}
{{ result.host }} - {{ result.status }}
{{ result.data.message or result.log }}
{% endfor %}{% endraw %}

Окремі Reports з одного запуску

Report Template може створити декілька Reports з одного запуску. Обгорни кожне тіло в [[WINHUB_REPORT title="..."]] і [[/WINHUB_REPORT]]. Це зручно для окремого листа на кожного користувача, клієнта або сервіс.

{% raw %}{% for username in usernames %}
[[WINHUB_REPORT title="Access letter - {{ username }}"]]
Привіт, {{ username }}.
Тут персональний текст.
[[/WINHUB_REPORT]]
{% endfor %}{% endraw %}

Правила виконання Windows Agent

  • Пиши PowerShell сумісний із 5.1, якщо не впевнений, що всі цілі мають новіший PowerShell.
  • Агент працює без участі користувача, часто як service/SYSTEM, тому не використовуй prompt-и, GUI-вікна, confirmations або інтерактивні installers.
  • Став $ErrorActionPreference = "Stop" і обробляй ризикові блоки через try/catch.
  • Для кирилиці і нормальних reports виставляй UTF-8 console output.
  • Пиши idempotent scripts: повторний запуск не має ламати вже налаштований стан.
  • Для локалізованих Windows груп краще використовувати SID: Administrators S-1-5-32-544, Remote Desktop Users S-1-5-32-555.
  • Використовуй повні шляхи до файлів та installer-ів. Тимчасові файли чисть після виконання.
  • Якщо Report Template має парсити результат, JSON має бути фінальним і коректним output.

Чого не робити

  • Не використовуй Read-Host, GUI installer-и або команди, які чекають на введення.
  • Не друкуй постійні паролі або токени, якщо цього не вимагає report/email workflow.
  • Не запускай destructive commands без перевірки конкретного target path або target object.
  • Не покладайся на current working directory; агент може стартувати з service/system paths.

Sharing і permissions

Вмикай Share with Team тільки для шаблонів, які безпечно давати операторам. Оператору також потрібен доступ до груп endpoint-ів і право run_tasks. Для reports є окремі права: view_reports, send_reports, edit_reports, delete_reports, view_sensitive_reports.
