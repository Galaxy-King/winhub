Action Script

Runs PowerShell or shell code on selected endpoints. Add variables as placeholders like {{ "{{admin_username}}" }}; WinHUB shows input fields before execution and API clients pass them in variables.

Metric Item

Runs as a reusable telemetry check. Return compact JSON or a scalar value; WinHUB stores the result as a host metric and can use it in trigger rules.

Report Template

Formats aggregated task results after execution. Use report templates for email bodies or review reports; they are selected in Post-Execution, not executed directly on hosts.

$TargetUser = "{{ "{{admin_username}}" }}"

# "variables": { "admin_username": "admin", "ticket_id": "INC-1001" }

Workspace Guide

Template authoring, execution rules, and sharing

Template Types

Action Script runs PowerShell, Bash, or shell code on selected nodes. Metric Item stores the result as a host metric for dashboards and triggers. Report Template formats aggregated task results and email reports; it is not executed directly on hosts.

Variables

Use placeholders like {{ "{{admin_username}}" }}. WinHUB detects them and asks for values before execution. API clients pass values in the variables object.

Code Rules

Write idempotent scripts when possible. Return clear text or JSON. Avoid interactive prompts because agents run unattended. Use UTF-8 output. For Windows commands, prefer PowerShell cmdlets and explicit error handling.

Sharing & Migration

Enable Share with Team to make a template available to operators with access to target groups. Use Export/Import in the Template Library to move templates between WinHUB servers.

Типи шаблонів

Action Script запускає PowerShell, Bash або shell-код на вибраних вузлах. Metric Item зберігає результат як метрику хоста для dashboard і trigger-ів. Report Template форматує зведені результати задач і email-звіти; напряму на хостах він не виконується.

Змінні

Використовуй плейсхолдери на кшталт {{ "{{admin_username}}" }}. WinHUB знайде їх і попросить значення перед запуском. API-клієнти передають значення в об’єкті variables.

Правила коду

Пиши скрипти так, щоб їх можна було безпечно запускати повторно. Повертай зрозумілий текст або JSON. Не використовуй інтерактивні prompt-и, бо агент працює без участі користувача. Для Windows-команд краще використовувати PowerShell cmdlets і явну обробку помилок.

Поширення і міграція

Увімкни Share with Team, щоб шаблон бачили оператори з доступом до потрібних груп. Export/Import у Template Library дозволяє переносити шаблони між WinHUB-серверами.
